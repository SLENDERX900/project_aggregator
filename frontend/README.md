# Module 3: Portfolio Showcase (Frontend)

## Overview
This is a modern, responsive React frontend that displays your AI-refined portfolio projects. It features clean project cards, technology filtering, and professional design.

## Features
- **Responsive Design**: Works perfectly on desktop, tablet, and mobile
- **Technology Filtering**: Click any tech tag to filter projects
- **Smart Sorting**: Projects sorted by stars and last updated
- **Professional UI**: Clean, modern design with smooth animations
- **GitHub Integration**: Direct links to repositories and GitHub profiles

## Tech Stack
- **React 18** - Modern UI framework
- **Vite** - Fast build tool and dev server
- **CSS Variables** - Easy theming and customization
- **GitHub Pages** - Free hosting (or any static host)

## Quick Start

### Prerequisites
- Node.js 16+ installed
- Portfolio data generated by `scripts/refiner.py`

### Installation
```bash
# Navigate to frontend directory
cd frontend

# Install dependencies
npm install

# Copy portfolio data (run this after updating your portfolio)
cp ../scripts/portfolio_ready.json public/portfolio_ready.json
```

### Development
```bash
# Start development server
npm run dev

# Visit http://localhost:5173/portfolio-aggregator/
```

### Build for Production
```bash
# Build static files
npm run build

# Preview build locally
npm run preview
```

## Deployment Options

### Option 1: GitHub Pages (Recommended - Free)
1. Push your code to GitHub
2. Enable GitHub Pages in repository settings
3. Set source to "GitHub Actions" or "Deploy from a branch"
4. For "Deploy from a branch", select:
   - Branch: `main` or `gh-pages`
   - Folder: `/` (root)

### Option 2: Vercel (Free)
1. Install Vercel CLI: `npm i -g vercel`
2. Run: `vercel`
3. Follow prompts to deploy

### Option 3: Netlify (Free)
1. Run: `npm run build`
2. Drag the `dist` folder to Netlify deploy page
3. Or connect your GitHub repository for automatic deployments

## Troubleshooting

### "Could not load portfolio_ready.json"
- Make sure you've run `scripts/refiner.py` first
- Check that `portfolio_ready.json` is in the `public/` folder
- Verify the JSON file is valid

### White screen or build errors
- Clear browser cache
- Delete `node_modules` and run `npm install` again
- Check console for specific error messages

### Deployment issues
- Ensure `base` in `vite.config.js` matches your repository name
- For GitHub Pages, the base should be `/your-repo-name/`
- For other hosts, set base to `/`

## Performance Tips
- Images are optimized automatically by Vite
- CSS is minified in production builds
- React is tree-shaken to reduce bundle size
- Consider lazy loading for large portfolios
